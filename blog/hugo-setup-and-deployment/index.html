Looking back at my first post, I said that I was using Hugo to build this blog. This time, I‚Äôll share how to create your own blog using Hugo and this is based on my own experience.

00 - Goals#

Below are the main goals of this post:

  • Installing Hugo
  • Using Hugo theme
  • Deploying Hugo site with Github.

01 - Prerequisites#

There are some prerequisites to accomplishing the previous goals:

  • A GitHub account
  • Git Bash for Windows users
  • Basics Git knowledge (commit, push, pull, creating repository, know what is local and remote repository)
  • Basics of CLI (cd, ls, pwd, mv, rm, mkdir)

I use Git Bash on Windows, so it has a bit Linux flavor

02 - Download Hugo#

Download Hugo executable binary at:

  • https://github.com/gohugoio/hugo/releases

Pick your Hugo version according to what OS you‚Äôre on. There is also the extended version, which you should use if you‚Äôre building your own theme or picking a theme that uses Sass/SCSS.

At this time, Hugo‚Äôs binary is not available in a system-wide (it‚Äôs not accessible outside the current directory).

So, we’ll make it accessible from anywhere by adding the binary location to what is known as the PATH variable.

For Windows users, create a folder at C:/ and name it bin

Once the folder is created, move your Hugo binary into it.

On the Edit window, click on New button and add C:\bin into that field.

After that, just hit all the OK button.

Open your Windows terminal and run hugo version. If it returns the same output, then go to the section 04.

03.02 Linux#

I know those who uses Linux probably already know how to üòÅ.

For Linux users, I would create it under /home/username/.local/[here]. Open up your terminal and type:

$ mkdir -p ~/.local/bin

Open your .bashrc or .zshrc file (it’s located at /home/username/.[zsh|bash]rc) with your favorite text editor, such as vim and simply add this line at the top of your .bashrc/.zshrc

PATH_HUGO='/home/username/.local/bin'
export PATH=$PATH_HUGO:$PATH

Reopen your terminal and run hugo version from any directory and see if it’s returns the version.

04 - Creating Your First Site#

To create a site you can type this command on your terminal

$ hugo new site [site-name]

I recommend you to create a site in a specific folder such as workspace.

Let’s create on site called my-blog.

$ hugo new site my-blog
Congratulations! Your new Hugo site is created in C:\Users\fahmi\Desktop\test\my-blog.

Just a few more steps and you're ready to go:

1. Download a theme into the same-named folder.
   Choose a theme from https://themes.gohugo.io/ or
   create your own with the "hugo new theme <THEMENAME>" command.
2. Perhaps you want to add some content. You can add single files
   with "hugo new <SECTIONNAME>\<FILENAME>.<FORMAT>".
3. Start the built-in live server via "hugo server".

Visit https://gohugo.io/ for quickstart guide and full documentation.

You can see Hugo creates a new folder called my-blog at C:\Users\fahmi\Desktop\test\my-blog.

05 - Run Your Site#

We can host the site in localhost simply by typing hugo server .

$ cd my-blog
$ hugo server

if you visit http://localhost:1313/ , It’s empty because we haven’t added any content yet.

06 - Installing Hugo Theme#

My blog use a theme called PaperMod, so l’ll be using that too here. But, first, let’s delete the previous my-blog and recreate it with the following command:

$ hugo new site my-blog -f yml

Enter the directory and initialize a git repository inside it

$ cd my-blog
$ git init

Then go to the themes folder and clone PaperMod theme there and Add the theme as submodule.

$ cd themes
$ git clone https://github.com/adityatelange/hugo-PaperMod PaperMod --depth=1
$ git submodule add https://github.com/adityatelange/hugo-PaperMod.git PaperMod

At the site root directory, replace/overwrite your config.yml with this but change the value of baseUrl and theme to this

baseURL: ""
theme: PaperMod

Test your theme with

$ hugo server

07 - Creating Your First Post#

We can create a new post with this command:

$ hugo new post/new-post.md

You can edit new-post.md after that, the file should be under my-blog/content/post/.

$ ls -l my-blog/content/post/
total 1
-rw-r--r-- 1 Fahmi FJ 197121 70 Jun 13 09:34 my-post.md

To see your post in the site, change the value draft from true to false:

---
title: "My Post"
date: 2021-06-13T09:34:43+07:00
draft: false
---

My first post

It should be on your site now.

08 - Deploying Site on GitHub#

First, you need to know that when you run hugo server, the site resources are served from memory. But, if you just type hugo, Hugo generates all your web files inside the my-blog/public folder. With that, you can simply upload all the files in the public folder into a GitHub repository.

Let’s do that in action. But before that, change your site’s base URL in config.yml to:

baseURL: "https://yourusername.github.io/my-blog/"

Run hugo at the site root directory to let Hugo re-generate the web source files at the public folder.

$ hugo

Then simply drag & drop all the files inside the my-blog/public/ to there.

From here, we learn how to deploy/host our Hugo site on GitHub. But, this is not an effective way to update your site, so let’s create a deployment script.

09 - Deployment Script#

Assuming you’re inside my-blog , then go to the public directory, do a clean up files and folders then initialize a git repository there.

$ cd public
$ rm -rf *
$ git init

Still inside the public directory, set the previously created my-blog repository as the remote repository and run git pull afterwards.

$ git remote add origin https://github.com/your-username/my-blog.git
$ git pull origin main

Return to the site root directory then add the public folder as git submodule.

$ cd ../
$ git submodule add https://github.com/your-username/my-blog.git public

Now let’s create a deploy script at the site root directory and name it as deploy.sh:

#!/bin/bash

echo -e "\033[0;32mDeploying blog to GitHub...\033[0m"

# Clean public folder
hugo --cleanDestinationDir

# Go to to public folder
cd public/

# Add untracked files, hide output
git add -A > /dev/null

# Generate a fixed commit message with date and time
msg="[`date "+%R %d-%h-%Y"]` Site update"
	
# Check for additional commit message
read -p "Add commit message: " add_msg
if [ "$add_msg" != "" ]
	then 
	msg="$msg - $add_msg"
fi

git commit -m "$msg"

# Deploy
git push -u origin main

# Go back to the root directory
cd ../

Let’s test it by creating a new post.

$ hugo new post/second-post.md
$ echo 'This is second post' >> content/post/second-post.md

Don’t forget to change the draft value to false!
